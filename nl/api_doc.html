Gratis Datalek-API

Controleer miljarden blootgestelde records met onze gratis RESTful API. Geen API-sleutel vereist.

Authenticatie

De meeste endpoints zijn openbaar. Domein-endpoints vereisen een API-sleutel.

Aanvraaglimiet

2 verzoeken per seconde per IP, plus limieten per uur en per dag per endpoint. Bekijk de limieten.

Antwoordformaat

Alle antwoorden retourneren JSON met standaard HTTP-codes.

Basis-URL

api.xposedornot.com

Swagger Documentatie API Speeltuin

Officiële SDK's

Gebruik onze officiële bibliotheken om XposedOrNot in uw applicaties te integreren met slechts een paar regels code.

Node.js / JavaScript

Installeer via npm: 

npm install xposedornot

Snel voorbeeld:

const { XposedOrNot } = require('xposedornot');

const xon = new XposedOrNot();

// Check email for breaches
const result = await xon.checkEmail('test@example.com');
console.log(result);

API Snelle Referentie Laatst bijgewerkt: 03-jun-2026

Hartelijk dank voor uw interesse in XposedOrNot. Het doel is om onze API zo toegankelijk en responsief mogelijk te houden, zodat het een waardevol hulpmiddel is voor iedereen. Direct gehost op Google Infrastructure en robuust gecached door Cloudflare, biedt de XposedOrNot API snelle antwoorden op alle vragen, ongeacht de oorsprong.

De XposedOrNot API volgt de principes van REST-architectuur. Het retourneert JSON-gecodeerde antwoorden en maakt gebruik van standaard HTTP-antwoordcodes. Voor de meeste API-routes zijn er geen authenticatievereisten, wat eenvoudige toegang en gebruiksgemak garandeert. Houd er echter rekening mee dat onze specifieke API-route voor het opvragen van domeingegevens een API-sleutelautorisation vereist. Deze maatregel is van kracht om te zorgen dat alleen geverifieerde eigenaren of autoriteiten van de opgegeven domeinen toegang hebben tot deze gevoelige informatie.

Om met de endpoints te experimenteren, kunt u de API Speeltuin gebruiken, waar u de XposedOrNot API interactief kunt verkennen.

De voorbeelden lezen: waarden tussen {accolades} zijn tijdelijke aanduidingen. Vervang elke tijdelijke aanduiding (inclusief de accolades) door uw eigen waarde. Bijvoorbeeld, {email} wordt test@example.com.

GET 1. API-aanroep: E-mailadres controleren op datalekken

Deze API controleert of een e-mailadres betrokken is geweest bij bekende datalekken. Het doorzoekt een uitgebreide database van datalekken en waarschuwt u als het e-mailadres risico loopt.


API-endpoint: 
https://api.xposedornot.com/v1/check-email/{email}
Parameters:
Parameters voor het check-email endpoint
Parameter In Vereist Type Beschrijving
email path Ja string E-mailadres om te controleren op datalekken.
details query Nee boolean Gedetailleerde lekgegevens opnemen in het antwoord. Standaard false (accepteert true/false, yes/no, 1/0).
Voorbeeld van succesvolle datalekdetectie: Wanneer een datalek wordt gedetecteerd, ontvangt u een JSON-antwoord zoals dit: 
 {
  "breaches": [
    [
      "Tesco",
      "KiwiFarms",
      "Vermillion",
      "Verified",
      "LizardSquad",
      "2fast4u",
      "Autotrader",

    ]
  ],
  "email": "test@example.com",

Het antwoord is in JSON-formaat, waardoor het eenvoudig te verwerken is met elke scripttaal. Hierdoor kunt u de gegevens gemakkelijk integreren in uw applicaties.

Als het e-mailadres niet wordt gevonden in een lekdatabase, ontvangt u het volgende JSON-antwoord: 
{"Error":"Not found","email":null}

GET 2. API-aanroep: Datalekanalyse voor e-mailadressen

Onze API biedt een diepgaande analyse van de datalekgeschiedenis van een e-mailadres. Het onthult wanneer en waar datalekken hebben plaatsgevonden en biedt essentiële analyses om de impact en ernst van deze incidenten te meten. Dit hulpmiddel is cruciaal voor het begrijpen van blootstellingsniveaus en het verbeteren van beveiligingsstrategieën.


API-endpoint: 
https://api.xposedornot.com/v1/breach-analytics?email={email}
Parameters:
Parameters voor het breach-analytics endpoint
Parameter In Vereist Type Beschrijving
email query Ja string Te analyseren e-mailadres.

De API reageert met twee mogelijke uitkomsten: succes of mislukking. Hieronder staan de belangrijkste onderdelen van een succesvol antwoord:


  • BreachesSummary: Krijg een snel overzicht van alle datalekken gekoppeld aan het e-mailadres, inclusief een lijst van getroffen sites. Ideaal voor een snelle controle van de lekgeschiedenis.

  • ExposedBreaches: Ontvang gedetailleerde informatie over elk datalek, inclusief de naam van de getroffen entiteit, beschrijving, domein, branche, risiconiveau, verwijzingen, blootgestelde gegevenstypen en het jaar en aantal blootgestelde records. Dit helpt bij het beoordelen van de specifieke details en ernst van elk datalek.

  • BreachMetrics: Dit onderdeel biedt analyses over de datalekken, zoals getroffen branches, wachtwoordsterkte, risicoscore, blootgestelde gegevenstypen en een jaarlijkse uitsplitsing. Deze gegevens zijn cruciaal voor het begrijpen van de volledige impact van de datalekken.

  • xposed_data: Krijg inzicht in de specifieke gegevenstypen die bij de lekken zijn blootgesteld, zoals namen, foto's, nationaliteiten, enz. Dit helpt bij het begrijpen van de aard en omvang van de blootstelling van persoonsgegevens.

  • PastesSummary: Geeft een overzicht van 'paste'-lekken (gegevensblootstelling op openbare pastebin-achtige diensten), inclusief een telling en de meest recente gebeurtenis. Het is een snelle manier om de blootstelling op deze platforms te meten.

  • ExposedPastes & PasteMetrics: Deze onderdelen geven gedetailleerde informatie en een jaarlijkse analyse van paste-lekken, wat een dieper inzicht biedt in blootstellingstrends door de tijd heen.

Deze uitgebreide suite van analysetools biedt een diepgaande analyse van de datalekgeschiedenis van elk e-mailadres en geeft de inzichten die nodig zijn voor een beter digitaal beveiligingsbeheer.

GET 3. API-aanroep om anoniem blootgestelde wachtwoorden te controleren

Dit endpoint controleert of een wachtwoord is verschenen in bekende datalekken zonder het wachtwoord zelf te onthullen. U berekent lokaal de SHA3 Keccak-512-hash van het wachtwoord en stuurt alleen de eerste 10 tekens, zodat het wachtwoord en de volledige hash uw machine nooit verlaten. Het antwoord geeft aan of er een overeenkomst is of niet. 


https://passwords.xposedornot.com/api/v1/pass/anon/{hash_prefix}
Hier is {hash_prefix} de eerste 10 tekens van de SHA3 Keccak-512-hash van het wachtwoord dat u wilt controleren.

Parameters:
Parameters voor het anonieme wachtwoordcontrole-endpoint
Parameter In Vereist Type Beschrijving
hash_prefix path Ja string Eerste 10 tekens van de SHA3 Keccak-512-hash van het te controleren wachtwoord.
Voorbeeld van JSON-uitvoer bij het vinden van een overeenkomende wachtwoordhash: 
 {
  "SearchPassAnon": {
    "anon": "808d63ba47",
    "char": "D:6;A:0;S:0;L:6",
    "count": "11999477",
    "wordlist": 0
  }
}

De API levert resultaten in JSON-formaat, wat meer informatief is dan een eenvoudig ja/nee. Deze gedetailleerde uitvoer maakt verdere analyse en verbetering van de uitgebreide lijst van in real-time blootgestelde wachtwoorden mogelijk.


Gids voor uitvoerstructuur:


  • anon: het door u opgezochte hash-voorvoegsel, teruggestuurd als bevestiging. Er wordt niets opgeslagen of verzonden dat uw wachtwoord kan onthullen.
  • char: een uitsplitsing van de samenstelling van het wachtwoord, in het formaat D:<cijfers>;A:<letters>;S:<speciaal>;L:<lengte> (zie de tabel hieronder).
  • count: het aantal keren dat dit wachtwoord is aangetroffen in de verzamelde blootgestelde datalekken.
  • wordlist: of het wachtwoord is gevonden in een bekende woordenlijst (1 = ja, 0 = nee).

Deze API is niet alleen nuttig voor het identificeren van blootgestelde wachtwoorden, maar helpt ook bij het ontwikkelen van sterkere en veiligere wachtwoordbeleid.


De volgende tabel legt de kenmerken in eenvoudige termen nader uit:

Uitsplitsing van wachtwoordkenmerken
Code Kenmerk Beschrijving
D Cijfers Aantal cijfers
A Letters Aantal letters
S Speciale tekens Aantal speciale tekens
L Lengte Lengte van het wachtwoord
Voor een uitgebreide lijst van alle blootgestelde websites, bezoek Blootgestelde websites in XON .

Een ander punt om op te letten is het gebruik van SHA3 Keccak-512 hashing voor het zoeken en opslaan van gegevens in XON. Traditionele hash-algoritmen zoals MD5 en SHA1 zijn momenteel verouderd en ook gezien het enorme aantal blootgestelde records heb ik gekozen voor het SHA3 Keccak-512-algoritme. Keccak-512-hashes zijn 128 tekens lang.

Bekijk het eenvoudige en gebruiksvriendelijke voorbeeldinlogscherm dat gebruik maakt van deze API.

POST 5. API-aanroep om alle datalekken voor geverifieerde domeinen op te vragen 

https://api.xposedornot.com/v1/domain-breaches/
Parameters:
Headers voor het domain-breaches endpoint
Parameter In Vereist Type Beschrijving
x-api-key header Ja string Uw domein API-sleutel. Het domein wordt afgeleid van de sleutel.
Content-Length header Ja 0 Dit endpoint accepteert geen aanvraagtekst, stel de inhoudslengte in op 0.
Dit is een POST-verzoek en vereist dat de geldige API-sleutel wordt opgenomen in de header met de sleutel 'x-api-key'. Het domein wordt afgeleid van de API-sleutel, dus dit endpoint accepteert geen aanvraagtekst en de inhoudslengte-header moet worden ingesteld op '0'.
** Heeft u nog geen sleutel? Zie hoe u een API-sleutel voor domein datalekgegevens kunt verkrijgen. Dit behandelt domeinverificatie en het ophalen van de sleutel via uw dashboard.

Voorbeeld-API-aanvraag met curl. Sla uw sleutel op in een omgevingsvariabele in plaats van deze hard te coderen, zodat deze buiten uw shellgeschiedenis en bronbeheer blijft: 
export XON_API_KEY="your_api_key_here"

curl -L -X POST \
  -H "x-api-key: $XON_API_KEY" \
  -H "Content-Length: 0" \
  https://api.xposedornot.com/v1/domain-breaches/
Het antwoord van de API is in JSON-formaat. De hoofdsleutel 'metrics' bevat details over het datalek. Hieronder volgt een beschrijving van elke subsleutel in 'metrics':

1.Breach_Summary: Dit veld geeft een samenvatting van het aantal datalekken per organisatie.
2.Breaches_Details: Dit is een array met gedetailleerde informatie over elk afzonderlijk datalek, inclusief de naam van de getroffen organisatie, het domein en het e-mailadres dat bij het datalek is betrokken.
3.Detailed_Breach_Info: Dit veld bevat een gedetailleerde samenvatting van de datalekken, inclusief de datum van het datalek, het logo van de organisatie, of het wachtwoord al dan niet in gevaar was, of het datalek doorzoekbaar is, het type blootgestelde gegevens, het totale aantal blootgestelde records en een beschrijving van het datalek.
4.Domain_Summary: Dit biedt een samenvatting van het aantal datalekken per domein.
5.Top10_Breaches: Dit veld biedt een lijst van de top-10 datalekken.
6.Yearly_Metrics: Dit veld biedt een jaarlijkse uitsplitsing van het aantal datalekken van 2010 tot het huidige jaar.

Foutafhandeling: Als de API-sleutel ongeldig is, reageert het endpoint met HTTP 401 en de volgende JSON: 
{
  "detail": "Invalid or missing API key"
}
Het veld detail bevat een beschrijving van de fout.

Vergeet niet om XON_API_KEY in te stellen op uw werkelijke API-sleutel voordat u een aanvraag doet bij het endpoint.

6. API-antwoordcodes


De XposedOrNot API gebruikt gangbare HTTP-antwoordcodes om het succes of de mislukking van een API-aanvraag aan te geven. In het algemeen: codes in het 2xx-bereik geven succes aan. Codes in het 4xx-bereik geven een fout aan die mislukte op basis van de verstrekte informatie (bijv. verkeerde parameters, onvoldoende zoekopties, verkeerde URL enz.). Codes in het 5xx-bereik geven een fout aan bij de server van XposedOrNot.

Met andere woorden, codes in het 4xx-bereik geven een client-zijde fout aan (uw aanvraag) en 5xx-codes geven een server-zijde fout aan (aan onze kant).

HTTP-antwoordcodes en hun betekenis
Code Beschrijving
200 Succes geeft JSON-antwoord terug
401 Ongeldige/ongeautoriseerde API-sleutel
404 Fout in invoer (geen gegevens gevonden)
429 Snelheidslimiet bereikt - vertragen a.u.b.
502/503 Server-zijde fout. Meld dit als u dit tegenkomt.
Opmerking: Zoekopdrachten voor wachtwoorden zijn niet beperkt door tijd of aantal.

Er geldt een drempel van 2 aanvragen per seconde voor elke API-aanvraag. Wij verzoeken u hier rekening mee te houden en opnieuw te proberen in een langzamere modus of met een pauze.

Herhaald misbruik of misbruik van deze API wordt automatisch geblokkeerd door xGuardian. Gebruik de API a.u.b. verantwoordelijk.
