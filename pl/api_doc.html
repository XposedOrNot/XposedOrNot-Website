Darmowe API wycieków danych

Sprawdzaj miliardy ujawnionych rekordów dzięki naszemu darmowemu API RESTful. Klucz API nie jest potrzebny.

Uwierzytelnianie

Większość punktów końcowych jest publiczna. Punkty końcowe domen wymagają klucza API.

Limit żądań

2 żądania na sekundę na adres IP, plus godzinowe i dzienne limity dla każdego punktu końcowego. Zobacz limity.

Format odpowiedzi

Wszystkie odpowiedzi zwracają JSON ze standardowymi kodami HTTP.

Bazowy URL

api.xposedornot.com

Skorzystaj z naszych oficjalnych bibliotek, aby zintegrować XposedOrNot ze swoimi aplikacjami za pomocą zaledwie kilku linijek kodu.

npm install xposedornot

const xon = new XposedOrNot();

// Check email for breaches

console.log(result);

Szybki przewodnik po API Ostatnia aktualizacja: 03-06-2026

Dziękujemy za zainteresowanie serwisem XposedOrNot. Naszym celem jest utrzymanie API tak dostępnego i responsywnego, jak to możliwe, aby było wartościowym narzędziem dla każdego. Hostowane bezpośrednio na infrastrukturze Google i solidnie buforowane przez Cloudflare, API XposedOrNot zapewnia szybkie odpowiedzi na wszystkie zapytania, niezależnie od miejsca ich pochodzenia.

API XposedOrNot jest zgodne z zasadami architektury REST. Zwraca odpowiedzi zakodowane w JSON i wykorzystuje standardowe kody odpowiedzi HTTP. Większość naszych tras API nie wymaga uwierzytelniania, co zapewnia łatwy dostęp i prostotę użycia. Pamiętaj jednak, że nasza specjalna trasa API służąca do zapytań o dane związane z domenami wymaga autoryzacji kluczem API. Ten środek zapewnia, że tylko zweryfikowani właściciele lub administratorzy wskazanych domen mają dostęp do tych wrażliwych informacji.

Aby poeksperymentować z punktami końcowymi, wypróbuj API Playground, gdzie możesz interaktywnie poznawać API XposedOrNot.

Jak czytać przykłady: wartości pokazane w {nawiasach klamrowych} to symbole zastępcze. Zastąp każdą z nich (wraz z nawiasami) własną wartością. Na przykład {email} staje się test@example.com.

HTTP 429 oznacza, że osiągnięto limit godzinowy lub dzienny: poczekaj na zresetowanie okna, zanim spróbujesz ponownie. Odpowiedź zawiera link do planów z wyższymi limitami.

Potrzebujesz wyższych limitów żądań?

Darmowe API jest przeznaczone do użytku osobistego i o niskim wolumenie, i pozostaje darmowe. Jeśli budujesz produkt oparty na danych o wyciekach lub potrzebujesz większej przepustowości, xonAPI+ oferuje płatne plany już od 5 USD/miesiąc z limitami do 25 000 żądań/minutę, dostępem przez klucz API i wsparciem komercyjnym. To te same dane o wyciekach, i to właśnie dzięki nim darmowy plan pozostaje darmowy.

GET 1. Wywołanie API: sprawdzanie wycieków danych adresu e-mail

To API sprawdza, czy adres e-mail pojawił się w znanych wyciekach danych. Przeszukuje obszerną bazę wycieków i ostrzega Cię, jeśli e-mail jest zagrożony.


Punkt końcowy API: 
https://api.xposedornot.com/v1/check-email/{email}
Parametry:
Parametry punktu końcowego check-email
Parametr W Wymagany Typ Opis
email path Tak string Adres e-mail do sprawdzenia pod kątem wycieków.
details query Nie boolean Dołącza szczegółowe informacje o wyciekach do odpowiedzi. Domyślnie false (akceptuje true/false, yes/no, 1/0).
Przykład pomyślnego wykrycia wycieku: Gdy wyciek zostanie wykryty, otrzymasz odpowiedź JSON podobną do tej: 
 {
  "breaches": [
    [
      "Tesco",
      "KiwiFarms",
      "Vermillion",
      "Verified",
      "LizardSquad",
      "2fast4u",
      "Autotrader",
      "MyRepoSpace",
      "SweClockers"
    ]
  ],
  "email": "test@example.com",
  "status": "success"
}
Odpowiedź jest w formacie JSON, dzięki czemu łatwo ją przetworzyć w dowolnym języku skryptowym. Pozwala to łatwo zintegrować dane z Twoimi aplikacjami.
Odpowiedź, gdy nie znaleziono wycieku:
Jeśli adres e-mail nie zostanie znaleziony w żadnej bazie wycieków, otrzymasz następującą odpowiedź JSON: 
{"Error":"Not found","email":null}

GET 2. Wywołanie API: analityka wycieków danych dla adresów e-mail

Nasze API oferuje dogłębną analizę historii wycieków danych adresu e-mail. Pokazuje, kiedy i gdzie doszło do wycieków, dostarczając kluczowej analityki do oceny wpływu i powagi tych incydentów. To narzędzie jest niezbędne do zrozumienia poziomu ekspozycji danych i wzmacniania strategii bezpieczeństwa.


Punkt końcowy API: 
https://api.xposedornot.com/v1/breach-analytics?email={email}
Parametry:
Parametry punktu końcowego breach-analytics
Parametr W Wymagany Typ Opis
email query Tak string Adres e-mail do przeanalizowania.

API odpowiada na dwa możliwe sposoby: sukcesem lub niepowodzeniem. Poniżej znajdują się kluczowe komponenty pomyślnej odpowiedzi:


  • BreachesSummary: Szybki przegląd wszystkich wycieków powiązanych z adresem e-mail, wraz z listą dotkniętych serwisów. Idealne do szybkiego sprawdzenia historii wycieków.

  • ExposedBreaches: Szczegółowe informacje o każdym wycieku, w tym nazwa podmiotu dotkniętego wyciekiem, opis, domena, branża, poziom ryzyka, odniesienia, typy ujawnionych danych oraz rok i liczba ujawnionych rekordów. Pomaga to ocenić szczegóły i powagę każdego wycieku.

  • BreachMetrics: Ten komponent dostarcza analitykę wycieków, taką jak dotknięte branże, siła haseł, ocena ryzyka, typy ujawnionych danych oraz podział roczny. Te dane są kluczowe do zrozumienia pełnego wpływu wycieków.

  • xposed_data: Wgląd w konkretne typy danych ujawnionych w wyciekach, takie jak imiona i nazwiska, zdjęcia, narodowości itd. Pomaga to zrozumieć charakter i zakres ekspozycji danych osobowych.

  • PastesSummary: Przegląd wycieków typu "paste" (ekspozycji danych w publicznych serwisach typu pastebin), wraz z liczbą i najnowszym wystąpieniem. To szybki sposób na ocenę ekspozycji na tych platformach.

  • ExposedPastes & PasteMetrics: Te komponenty dostarczają szczegółowych informacji i rocznej analizy wycieków typu paste, pozwalając głębiej zrozumieć trendy ekspozycji w czasie.

Ten kompleksowy zestaw narzędzi analitycznych oferuje dogłębny wgląd w historię wycieków danych dowolnego e-maila, dostarczając informacji potrzebnych do lepszego zarządzania bezpieczeństwem cyfrowym.

GET 3. Wywołanie API do anonimowego sprawdzania ujawnionych haseł

Ten punkt końcowy sprawdza, czy hasło pojawiło się w znanych wyciekach, bez ujawniania samego hasła. Obliczasz lokalnie hash SHA3 Keccak-512 hasła i wysyłasz tylko jego pierwsze 10 znaków, więc hasło i pełny hash nigdy nie opuszczają Twojego komputera. Odpowiedź wskazuje trafienie lub jego brak. 


https://passwords.xposedornot.com/api/v1/pass/anon/{hash_prefix}
Tutaj {hash_prefix} to pierwsze 10 znaków hasha SHA3 Keccak-512 hasła, które chcesz sprawdzić.

Parametry:
Parametry punktu końcowego anonimowego sprawdzania haseł
Parametr W Wymagany Typ Opis
hash_prefix path Tak string Pierwsze 10 znaków hasha SHA3 Keccak-512 hasła do sprawdzenia.
Przykładowy wynik JSON po pomyślnym znalezieniu pasującego hasha hasła: 
 {
  "SearchPassAnon": {
    "anon": "808d63ba47",
    "char": "D:6;A:0;S:0;L:6",
    "count": "11999477",
    "wordlist": 0
  }
}

API dostarcza wyniki w formacie JSON, który jest bardziej informacyjny niż proste tak/nie. Ten szczegółowy wynik umożliwia dalszą analizę i rozbudowę obszernej listy ujawnionych haseł w czasie rzeczywistym.


Przewodnik po strukturze wyniku:


  • anon: odesłany z powrotem prefiks hasha, o który pytasz. Nic, co mogłoby ujawnić Twoje hasło, nie jest przechowywane ani przesyłane.
  • char: rozkład składu hasła w formacie D:<digits>;A:<alphabets>;S:<special>;L:<length> (zobacz tabelę poniżej).
  • count: liczba wystąpień tego hasła w zgromadzonych ujawnionych wyciekach danych.
  • wordlist: czy hasło znaleziono w znanej liście słów (1 = tak, 0 = nie).

To API jest przydatne nie tylko do identyfikowania ujawnionych haseł, ale pomaga też tworzyć silniejsze, bezpieczniejsze polityki haseł.


Poniższa tabela wyjaśnia nieco więcej o tych cechach w prostych słowach:

Pełną listę wszystkich ujawnionych stron znajdziesz na Ujawnione strony w XON .

Warto też zwrócić uwagę na użycie hashowania SHA3 Keccak-512 do wyszukiwania i przechowywania danych w XON. Tradycyjne algorytmy hashujące, takie jak MD5 i SHA1, są obecnie przestarzałe, a biorąc pod uwagę ogromną liczbę ujawnionych rekordów, zdecydowałem się na algorytm SHA3 Keccak-512. Hashe Keccak-512 mają 128 znaków długości.

Sprawdź prosty i łatwy w użyciu przykładowy ekran logowania, korzystający z tego API.

https://api.xposedornot.com/v1/domain-breaches/
Parametry:
Nagłówki punktu końcowego domain-breaches
Parametr W Wymagany Typ Opis
x-api-key header Tak string Twój klucz API domeny. Domena jest wyprowadzana z klucza.
Content-Length header Tak 0 Ten punkt końcowy nie przyjmuje treści żądania, więc ustaw długość treści na 0.
To jest żądanie POST i wymaga dołączenia w nagłówku ważnego klucza API pod kluczem 'x-api-key'. Domena jest wyprowadzana z klucza API, więc ten punkt końcowy nie przyjmuje żadnej treści żądania, a nagłówek długości treści powinien być ustawiony na '0'.
** Nie masz jeszcze klucza? Zobacz, jak uzyskać klucz API dla danych o wyciekach domen. Opisano tam weryfikację domeny i pobieranie klucza z Twojego panelu.

Przykładowe żądanie API z użyciem curl. Przechowuj swój klucz w zmiennej środowiskowej zamiast wpisywać go na stałe, aby nie trafił do historii powłoki ani do systemu kontroli wersji: 
export XON_API_KEY="your_api_key_here"

curl -L -X POST \
  -H "x-api-key: $XON_API_KEY" \
  -H "Content-Length: 0" \
  https://api.xposedornot.com/v1/domain-breaches/
Odpowiedź API jest w formacie JSON. Główny klucz 'metrics' zawiera szczegóły wycieku. Poniżej znajduje się opis każdego podklucza w 'metrics':

1.Breach_Summary: To pole zawiera podsumowanie liczby wycieków na organizację.
2.Breaches_Details: To tablica zawierająca szczegółowe informacje o każdym pojedynczym wycieku, w tym nazwę organizacji dotkniętej wyciekiem, domenę oraz adres e-mail powiązany z wyciekiem.
3.Detailed_Breach_Info: To pole zawiera szczegółowe podsumowanie wycieków, w tym datę wycieku, logo organizacji, informację, czy hasło było zagrożone, czy wyciek jest przeszukiwalny, typ ujawnionych danych, łączną liczbę ujawnionych rekordów oraz opis wycieku.
4.Domain_Summary: Zawiera podsumowanie liczby wycieków na domenę.
5.Top10_Breaches: To pole zawiera listę 10 największych wycieków.
6.Yearly_Metrics: To pole zawiera roczny podział liczby wycieków od 2010 roku do bieżącego roku.

6. Kody odpowiedzi API


API XposedOrNot używa konwencjonalnych kodów odpowiedzi HTTP do wskazania sukcesu lub niepowodzenia żądania API. Ogólnie: kody z zakresu 2xx oznaczają sukces. Kody z zakresu 4xx oznaczają błąd wynikający z przekazanych informacji (np. błędne parametry, niewystarczające opcje zapytania, błędny adres URL itp.). Kody z zakresu 5xx oznaczają błąd po stronie serwera XposedOrNot.

Innymi słowy, kody z zakresu 4xx wskazują błąd po stronie klienta (Twoje żądanie), a kody 5xx wskazują błąd po stronie serwera (u nas).

Kody odpowiedzi HTTP i ich znaczenia
Kod Opis
200 Sukces, zwrócona zostanie odpowiedź JSON
401 Nieprawidłowy/nieautoryzowany klucz API
404 Błąd w danych wejściowych ( nie znaleziono danych )
429 Osiągnięto limit szybkości, czas zwolnić
502/503 Błąd po stronie serwera. Zgłoś go, jeśli go napotkasz.
Uwaga: Zapytania wyszukiwania haseł nie są ograniczone czasem ani liczbą.

Obowiązuje próg 2 zapytań na sekundę dla każdego żądania API. Miej to na uwadze i w razie potrzeby ponów próbę w trybie wolniejszym lub z użyciem uśpienia.
