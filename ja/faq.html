Q. XON には何が保存され、何が保存されないのですか?

XusedOrNot は、既知のデータ侵害によって個人情報が漏洩していないかどうかをユーザーが確認できるデータ侵害監視サービスです。私は、サービスに何が保存され、何が保存されないかについてユーザーに対して透明性を保ちたいと考えています。

その質問に答えるために、当社はユーザーのパスワードや個人を特定できる情報 (PII) を XON に保存しません。ユーザーが当社の検索エンジンに電子メール アドレスまたはドメイン名を入力すると、当社は既知のデータ侵害のデータベースをチェックして、その電子メールまたはドメインが過去の侵害に関与していないかどうかを確認します。一致する場合は、そのインシデントに関して当社が把握している追加の詳細情報とともに、ユーザーの電子メールまたはドメインが関与した特定の侵害に関する情報をユーザーに提供します。

当社は、検索されたメールアドレスやドメイン名、検索日など、ユーザーの検索に関する情報を保存せず、ユーザーのデータのプライバシーとセキュリティを確保するための措置を講じています。

要約すると、XON はユーザーのパスワードや PII を保存しませんが、サービスを向上させる目的でユーザー検索に関するいくつかの基本情報を保存します。

Q. 電子メールのデータ侵害リスク スコアはどのように計算すればよいですか?

リスクスコアの計算式とは何ですか?

のリスクスコアの計算式 ユーザーの電子メールに関連するデータ侵害の数と重大度、最後の侵害からの経過時間、およびユーザーのパスワードの強度に基づいて、ユーザーのアカウントのリスクを計算します。

式は次のとおりです。
リスクスコア = (A x W1) + (B x W2) + (C x W3) + (D x W4)

どこ：
A = ユーザーの電子メール アドレスに関連するデータ侵害の数
B = ユーザーの電子メール アドレスに関連するデータ侵害の重大度
C = ユーザーの電子メール アドレスに関連する最後の侵害からの時間
D = パスワード強度アルゴリズムに基づくパスワード強度

そして：

W1 = 係数 A の重み
W2 = 係数 B の重み
W3 = 係数 C の重み
W4 = 係数 D の重み

たとえば、各要素に次の重みを割り当てるとします。

W1 = 4 (データ侵害の数が最も重要な要素と考えられます)
W2 = 2 (データ侵害の重大度も重要ですが、侵害の数ほど重要ではありません)
W3 = 1 (最後の侵害からの時間は依然として重要ですが、他の要素ほど重要ではありません)
W4 = 3 (パスワードの強度も重要ですが、侵害の数ほど重要ではありません)

次に、次の式を使用してユーザーのリスク スコアを計算できます。

リスクスコア = (A x 4) + (B x 2) + (C x 1) + (D x 3)

たとえば、ユーザーが 5 件のデータ侵害に関与しており、そのうち 2 件は重大度の高い侵害で、最後の侵害は 6 か月前に発生しており、強力なパスワードを持っている場合、そのリスク スコアは次のようになります。

リスクスコア = (5 x 4) + (2 x 2) + (6/12 x 1) + (1 x 3) = 31

リスクスコアはどのように計算されますか?

各要素にその重みを乗算し、その結果を加算してリスク スコアを計算します。リスク スコアは信号機のように視覚的に表示され、緑色は低リスク、黄色は中程度のリスク、赤色は高リスクを示します。

リスクスコアの計算式ではどのような要素が考慮されますか?

リスク スコアの計算式で考慮される要素は、ユーザーの電子メールに関連するデータ侵害の数と重大度、最後の侵害からの経過時間、およびユーザーのパスワードの強度です。
