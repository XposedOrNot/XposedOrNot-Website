P: O que é XposedOrNot (XON)?

XposedOrNot (XON) é uma ferramenta prática e engenhosa projetada para permitir que você verifique 🔍 se seus dados pessoais foram implicados em uma violação de dados. Originalmente, esta iniciativa nasceu do meu profundo interesse pela ciência forense e pela análise de violações de dados. A constatação do grande volume de informação sensível exposta e dos potenciais danos que poderia causar levou-me rapidamente a transformar este interesse num serviço público. Fiquei motivado a oferecer este serviço a todos gratuitamente, tornando-o acessível a todos os que queiram proteger os seus dados pessoais e privacidade.

OXposedOrNot A plataforma ajuda as pessoas a descobrir se seus e-mails ou dados pessoais foram comprometidos em uma violação de dados. Esse conhecimento pode impulsionar os usuários a proteger suas contas com medidas como alterações de senha e monitoramento vigilante das contas.

Além de apenas verificar e-mails e senhas, o site também facilita aos usuários o exame minucioso de informações sobre violação de dados em umnível de domínio ou subdomínio. Para aqueles que conseguem confirmar a propriedade do domínio, têm acesso a uma análise detalhada das violações de dados que afetaram os seus utilizadores.

Uma adição interessante ao nosso conjunto de ferramentas é oPainel CXO. Este recurso é particularmente útil para empresas e corporações que gerenciam vários domínios. O Painel CXO oferece uma visão unificada de violações de dados e todas as análises associadas. Esta perspectiva consolidada pode simplificar significativamente a complexa tarefa de monitorização de violações em vários domínios, capacitando as organizações a compreender e responder a incidentes de segurança de forma mais eficiente e eficaz.

Xposed (repositório de página única) contém detalhes sobre todas as violações de dados carregadas no XposedOrNot. Este repositório foi projetado para ser visualmente envolvente, simplificando a compreensão dos aspectos exclusivos de cada violação. Para pessoas que preferem uma lista simples de linhas em uma tabela, você pode consultarpágina de violações.

Na XposedOrNot, eu obtenho a maioria dos nossos dados expostos de violações que estão disponíveis na Internet. P. O que está armazenado e o que não está armazenado no XON?

XposedOrNot é um serviço de monitoramento de violação de dados que permite aos usuários verificar se informações pessoais foram expostas em alguma violação de dados conhecida. Quero ser transparente com nossos usuários sobre o que está e o que não está armazenado em nosso serviço.

Para responder a essa pergunta, não armazenamos nenhuma senha de usuário ou informação de identificação pessoal (PII) no XON. Quando um usuário insere seu endereço de e-mail ou nome de domínio em nosso mecanismo de pesquisa, verificamos nosso banco de dados de violações de dados conhecidas para ver se esse e-mail ou domínio esteve envolvido em alguma violação anterior. Se houver uma correspondência, forneceremos ao usuário informações sobre as violações específicas em que seu e-mail ou domínio esteve envolvido, juntamente com quaisquer detalhes adicionais que tivermos sobre o incidente.

Não armazenamos informações sobre pesquisas de usuários, como e-mail ou nome de domínio pesquisado e a data da pesquisa, e tomamos medidas para garantir a privacidade e segurança dos dados de nossos usuários.

Em resumo, a XON não armazena nenhuma senha de usuário ou PII, mas armazenamos algumas informações básicas sobre pesquisas de usuários com o objetivo de melhorar nosso serviço.

XON também tem a capacidade de verificarsenhas expostas . Este serviço utiliza o Algoritmo de hash SHA3-keccack 512 para converter as senhas coletadas em hashes unidirecionais no armazenamento. Com as tecnologias atuais disponíveis, é altamente improvável que alguém consiga reverter facilmente esses hashes SHA-3 Keccak. Isso garante o mais alto nível de segurança para hashes armazenados.

Por favor, cheque o exemplo de página de login, fazendo uso da API de senhas XON. Isso pode ajudar muitos usuários, impedindo-os de reutilizar senhas antigas e expostas em linha comDiretrizes do NIST.

P. Como classifico as violações de dados?

As violações de dados são atualmente classificadas da seguinte forma:

Violação de dados: A violação de dados é a liberação intencional ou não intencional de informações seguras ou privadas/confidenciais para um ambiente não confiável. Sites ou aplicativos online que têm seus dados violados por recursos externos ou internos e expostos/violados na web para acesso não autorizado.

Lista de combinação: A lista de combinação é um arquivo de texto que contém uma lista de nomes de usuários e senhas em um formato consistente. Os Combolists devem ser legíveis por máquina para que possam ser usados como entrada para ferramentas que automatizarão solicitações de autenticação para um site ou API.

Tags usadas em violações expostas :
Verificado: violações que são verificadas individualmente e confirmadas como violações de dados autênticas.
Não confiável: Violações que não são verificadas e não confirmadas.
Pesquisável: Violações ou ComboLists que podem ser pesquisadas publicamente com um endereço de e-mail.
Site sensível: Violações que não podem ser pesquisadas publicamente considerando a sensibilidade dos dados expostos.

P. Como calculo a pontuação de risco de violação de dados de um e-mail?

Qual é a fórmula de pontuação de risco?

Ofórmula de pontuação de risco calcula o risco da conta de um usuário com base no número e na gravidade das violações de dados associadas ao e-mail do usuário, no tempo desde a última violação e na força da senha do usuário.

A fórmula é:
Pontuação de risco = (A x W1) + (B x W2) + (C x W3) + (D x W4)

onde:
A = número de violações de dados associadas ao endereço de e-mail do usuário
B = gravidade das violações de dados associadas ao endereço de e-mail do usuário
C = tempo desde a última violação associada ao endereço de e-mail do usuário
D = força da senha com base em um algoritmo de força da senha

e:

W1 = peso para o fator A
W2 = peso para o fator B
W3 = peso para o fator C
W4 = peso para o fator D

Por exemplo, digamos que atribuímos os seguintes pesos a cada fator:

W1 = 4 (o número de violações de dados é considerado o fator mais importante)
W2 = 2 (a gravidade das violações de dados também é importante, mas não tanto quanto o número de violações)
W3 = 1 (o tempo desde a última violação ainda é importante, mas menos que os outros fatores)
W4 = 3 (a força da senha também é importante, mas não tanto quanto o número de violações)

Poderíamos então usar a seguinte fórmula para calcular a pontuação de risco de um usuário:

Pontuação de risco = (A x 4) + (B x 2) + (C x 1) + (D x 3)

Por exemplo, se um usuário esteve envolvido em 5 violações de dados, sendo 2 delas violações de alta gravidade, e a última violação ocorreu há 6 meses, e tem uma senha forte, sua pontuação de risco seria:

Pontuação de risco = (5 x 4) + (2 x 2) + (6/12 x 1) + (1 x 3) = 31

Como a pontuação de risco é calculada?

Cada fator é multiplicado pelo seu peso e os resultados são somados para calcular a pontuação de risco. A pontuação de risco será exibida visualmente como um sistema de semáforos, com verde indicando baixo risco, amarelo indicando risco moderado e vermelho indicando alto risco.

Que fatores são considerados na fórmula da pontuação de risco?

Os fatores considerados na fórmula de pontuação de risco são o número e a gravidade das violações de dados associadas ao email do usuário, o tempo desde a última violação e a força da senha do usuário.
