问：XON 中存储了什么，不存储什么？

XposedOrNot 是一项数据泄露监控服务，允许用户检查个人信息是否在任何已知的数据泄露中被泄露。我希望对我们的用户公开我们的服务中存储的内容和未存储的内容。

为了回答这个问题，我们不会在 XON 中存储任何用户密码或个人身份信息 (PII)。当用户在我们的搜索引擎中输入他们的电子邮件地址或域名时，我们会检查已知数据泄露数据库，看看该电子邮件或域名是否涉及过去的任何泄露事件。如果匹配，我们会向用户提供有关其电子邮件或域名所涉及的特定违规行为的信息，以及我们掌握的有关该事件的任何其他详细信息。

我们不存储有关用户搜索的信息，例如搜索的电子邮件或域名以及搜索日期，并且我们采取措施确保用户数据的隐私和安全。

总之，XON 不会存储任何用户密码或 PII，但我们会存储一些有关用户搜索的基本信息，以改进我们的服务。

XON 还具有检查能力暴露的密码 。该服务利用 SHA3-kecack 512 哈希算法 用于将收集到的密码转换为存储中的单向哈希值。凭借当前可用的技术，某人不太可能轻松逆转这些 SHA-3 Keccak 哈希值。这确保了存储的哈希值的最高级别的安全性。

请检查 示例登录页面， 使用 XON 密码 API。这可以帮助很多用户，防止他们重复使用旧的和暴露的密码NIST 指南。

问：如何对数据泄露进行分类？

数据泄露目前分为以下几类：

数据泄露： A 数据泄露 是有意或无意地将安全或私人/机密信息发布到不受信任的环境。其数据被外部或内部资源泄露并在网络上暴露/泄露以供未经授权的访问的网站或在线应用程序。

组合列表： A 组合列表 是一个文本文件，包含格式一致的用户名和密码列表。组合器应该是机器可读的，因此它们可以用作工具的输入，这些工具将自动向网站或 API 发出身份验证请求。

标签用于 暴露的违规行为 :
已验证：经过单独验证并确认为真实数据泄露的泄露。
不值得信赖：未经核实和确认的违规行为。
可搜索：可以使用电子邮件地址公开搜索的违规行为或组合列表。
敏感部位：考虑到所暴露数据的敏感性，无法公开搜索的违规行为。

问：如何计算电子邮件的数据泄露风险评分？

风险评分公式是什么？

这风险评分公式 根据与用户电子邮件相关的数据泄露的数量和严重性、自上次泄露以来的时间以及用户密码的强度来计算用户的帐户风险。

公式为：
风险评分 = (A x W1) + (B x W2) + (C x W3) + (D x W4)

在哪里：
A = 与用户电子邮件地址相关的数据泄露次数
B = 与用户电子邮件地址相关的数据泄露的严重程度
C = 自上次与用户电子邮件地址相关的违规事件以来的时间
D = 基于密码强度算法的密码强度

和：

W1 = 因素 A 的权重
W2 = 因素 B 的权重
W3 = 因素 C 的权重
W4 = 因素 D 的权重

例如，假设我们为每个因素分配以下权重：

W1 = 4（数据泄露次数被认为是最重要的因素）
W2 = 2（数据泄露的严重性也很重要，但不如泄露的数量）
W3 = 1（自上次违规以来的时间仍然很重要，但不如其他因素重要）
W4 = 3（密码强度也很重要，但不如泄露次数那么重要）

然后我们可以使用以下公式来计算用户的风险评分：

风险评分 = (A x 4) + (B x 2) + (C x 1) + (D x 3)

例如，如果用户涉及 5 次数据泄露，其中 2 次为高严重性泄露，最后一次泄露发生在 6 个月前，并且拥有强密码，则其风险评分将为：

风险评分 = (5 x 4) + (2 x 2) + (6/12 x 1) + (1 x 3) = 31

风险评分是如何计算的？

每个因素乘以它的权重，然后将结果相加来计算风险评分。风险评分将以红绿灯系统的形式直观显示，绿色表示低风险，黄色表示中等风险，红色表示高风险。

风险评分公式考虑了哪些因素？

风险评分公式中考虑的因素包括与用户电子邮件相关的数据泄露的数量和严重性、自上次泄露以来的时间以及用户密码的强度。
